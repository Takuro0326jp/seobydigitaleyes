このページは、公開レスポンス・HTML・リンク・リソース・一般的な公開URL探索から見える範囲のみで判断する設計です。 WAF、IDS/IPS、サーバ内設定などは外部から確証が取れないため別診断領域として切り分けています。

推奨アクション

1. X-Frame-Options または frame-ancestors を設定する High Header 比較的短時間

管理画面・ログイン画面・フォーム系ページは特に優先です。CSP の frame-ancestors を使うなら、X-Frame-Options とあわせて整理するとよいです。

2. 認証系 Cookie に Secure / HttpOnly / SameSite を付与する High Cookie

セッションIDやログイン維持Cookieは最優先です。外部から見ても対策有無が分かりやすく、スコア改善効果も大きい領域です。

3. Content-Security-Policy を段階導入する Medium XSS

まずは report-only から始め、使用中の外部タグを洗い出してから本適用する流れが安全です。

4. Server / X-Powered-By / 不要公開ファイルの露出を減らす Medium Exposure

技術情報の露出は直接侵害ではなくても、攻撃の足がかりになります。デプロイ時に backup、old、test、phpinfo 系ファイルが出ていないかも同時に確認します。
