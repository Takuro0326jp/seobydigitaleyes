Application Architecture

Frontend
  • public/*.html（各機能ページ）
  • public/js/*.js（機能別スクリプト）
  • public/css/tailwind.css + style.css
  • header.js（共通ナビ）
  • PWA対応（manifest + sw.js）
Backend (Express)
  • server.js（エントリーポイント）
  • routes/auth.js（認証）
  • routes/scan.js + scans.js（SEO診断）
  • routes/gsc.js（Search Console）
  • routes/ads.js / meta.js（広告）
  • routes/heatmap.js（ヒートマップ）
  • routes/strategy.js（AIアドバイス）
  • services/（共通ロジック）
Database
  • MySQL（mysql2）
  • 主要テーブル：
  • users / companies
  • scans / scan_pages / scan_links
  • gsc_tokens / google_ads_tokens
  • heatmap_sites / heatmap_events
  • action_items

SEO Scan Flow (Puppeteer クロール)

ユーザーが対象URLを入力 → POST /api/scans/start
↓
services/crawlQueue.js + scanCrawl.js
Puppeteer でページクロール・レンダリング
内部リンク抽出・再帰クロール
↓
解析項目
タイトル/メタ 見出し構造 画像alt リンク構造 セキュリティ モバイル対応 サイトマップ
↓
MySQL に保存 → GET /api/scans/result/:id で結果取得
↓
Gemini AI (services/actionItemGeneration.js)
スキャン結果をもとに改善アクションアイテム生成

Auth & Access Control

セッション認証 (services/session.js)
  • ・JWT Cookie ベース認証
  • ・/api/auth/* でログイン/ログアウト
  • ・ロール: admin / user
アクセス制御 (services/accessControl.js)
  • ・会社 (company) 単位でのデータ分離
  • ・ユーザー URL アクセス権管理
  • ・ヒートマップ: site_key 認証
レート制限
  • ・/api/* — 400 req/分
  • ・express-rate-limit 使用
セキュリティ
  • ・helmet.js（HTTPヘッダー強化）
  • ・CORS: tracker は cross-origin 許可
  • ・nginx: HTTPS終端（本番）
